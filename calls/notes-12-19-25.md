## Updated notes paraphrased from YK

* The edge-differential-privacy math itself is straightforward. If you view the **random walk on the transaction graph as the DP “algorithm”**, then removing edges corresponds to the adversary learning that certain coin links are impossible. The DP parameter ( \varepsilon ) bounds how much the probability of reaching any absorber (final coin) can change under such edge removals.

* The real open question is **interpretational**, not mathematical. In standard DP, you fix epsilon  first and design an algorithm that is both useful and privacy-preserving. Here, the algorithm (the random walk) and the data (the transaction graph) are fixed, and epsilon becomes something to **estimate**, not choose.

* Entropy and epsilon measure different things. Entropy reflects the *size* of the anonymity set (how large the volume is), while epsilon reflects its *stability* (how much that volume collapses when information is revealed). A large anonymity set can still be fragile.

* Example: a two-party CoinJoin that bridges two otherwise disjoint transaction graphs. If that CoinJoin is deanonymized, the graph splits and the anonymity set can drop by half, even though both sides were large. This corresponds to a bounded but significant epsilon.

* In contrast, if a single user later links pre- and post-mix coins (e.g., address reuse), the probability of some absorbers can drop to zero from a single edge removal. In that case, ( \varepsilon ) is effectively unbounded, even though the anonymity-set size only shrinks by one.

* On Samurai CoinJoins: with an honest coordinator and genuinely random peer selection, they actually have a much better chance of forming expander-like structure than Wasabi-style graphs, which suffer from strong temporal assortativity. The earlier statement was an oversimplification.

* When estimating degree sequences, the focus is **not the full transaction graph**, but only the **randomized subgraph**. Each transaction contains a mix of “regular” inputs and “exceptional” inputs whose spacings suggest randomized selection.

* By observing these spacings, one can estimate the **size of the UTXO selection pool**, which indirectly constrains the degree sequence of the randomized subgraph. This is neither a pure Erdős–Rényi-style independent edge model nor a fixed-degree model.

* A reasonable model is that the degree sequence evolves as a **Markov process**, making the observed degrees a **hidden Markov model**. That said, this can also be handled empirically by simulation, and it is acceptable to err on the side of **underestimating** anonymity or path counts.
